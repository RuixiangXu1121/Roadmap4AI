## 第一阶段：重构数据契约 (Data Schema) —— 定义“综述级”输出标准

在写任何检索代码之前，必须先定义这个模块要向下游（如 TCM-Agent 的网络可视化或报告生成模块 ）输出什么样的数据。这是整个多智能体协作的基石。



- **核心动作：** 放弃输出非结构化的“总结文本”，改为定义严格的 JSON/Pydantic 数据模型。

- **设计思路（多维证据矩阵）：**

  每一条被验证的网络药理学关系（Edge），必须包含以下维度：

  - `Source_Node` (如：黄芩素) & `Target_Node` (如：TNF-α)
  - `Action_Direction` (明确的上调/下调/抑制/激活)
  - `Confidence_Score` (综合置信度得分，0-100)
  - `Evidence_Distribution` (支持该结论的文献分布：体内实验 X 篇，体外实验 Y 篇，纯计算 Z 篇)
  - `Critical_Parameters` (提取出的有效给药剂量范围、细胞系或动物模型类型)
  - `Scientific_Conflict` (是否存在学术争议，如“A文献说抑制，B文献说无效”)

## 第二阶段：升级感知层 —— 多路召回与全文语义切片 (Deep Context Extraction)

打破目前仅看 5 篇 PubMed 摘要的瓶颈 。优秀的综述必须基于大量的、深度的原文阅读。

- **步骤 2.1：多源动态召回 (Multi-Source Recall)**
  - **思路：** 接入 PubMed API (获取英文摘要)、PMC API (获取开源英文全文 XML)。未来可扩展 RPA 工具抓取 CNKI (知网) 的中文核心期刊。
  - **动作：** 针对靶点组合，一次性拉取 50-100 篇文献的元数据。使用轻量级 Embedding 模型（如 BGE-M3）计算文献与研究问题的“余弦相似度”，截取 Top 30 篇进入精读。
- **步骤 2.2：上下文感知切片 (Context-Aware Chunking)**
  - **思路：** 网络药理学的硬核证据（剂量、P值、Western Blot 结果）都在正文里。
  - **动作：** 引入视觉解析工具（如 Marker 或 Nougat）将 PDF/XML 转为 Markdown。利用正则匹配，**强制剥离**“引言 (Introduction)”和“讨论 (Discussion)”，**仅保留**“材料与方法 (Methods)”和“结果 (Results)”段落。这能从物理上阻断大模型的“幻觉”来源。

## 第三阶段：重构大脑 —— 引入多智能体辩论与共识机制 (Debate & Consensus Engine)

这是该模块具备“综述级认知能力”的灵魂所在。大模型很容易“脑补”因果关系，必须引入对抗机制来提纯数据。

- **步骤 3.1：提取者智能体 (Proposer Agent)**
  - **职责：** 乐观的数据挖掘者。阅读切片后的方法与结果段落，尽可能多地提取出 `(药物-靶点-剂量-模型)` 关系。
- **步骤 3.2：批判者智能体 (Critic/Reviewer Agent)**
  - **职责：** 严苛的同行评审。它的 Prompt 中预设了“找茬”逻辑：检查提取者给出的证据是否具有统计学显著性？是否只是联合用药的副产物？是否是阴性结果（明确表述为无影响）？
- **步骤 3.3：共识计算与冲突标记 (Consensus Routing)**
  - **职责：** 当提取者和批判者对多篇文献进行处理后，该节点负责“求同存异”。如果 8 篇文献一致支持，则提高置信度；如果发现不同文献结论相反，则触发 `Conflict_Flag = True`，并记录冲突原因（如：剂量依赖性差异）。

## 第四阶段：规则沉淀 —— 证据效力硬编码与沙盒输出 (Evidence Grading)

将大模型的不确定性转化为确定性的数学得分，为 TCM-Agent 的整体评估提供客观依据。

- **步骤 4.1：基于规则的定级 (Rule-based Scoring)**
  - **思路：** 不让大模型凭感觉打分。只要提取者抓取到了“小鼠模型/In vivo”标签，代码层直接赋高分（如 80分）；如果抓取到“细胞系/In vitro”，赋中等分（如 60分）；如果是“网络预测/In silico”，赋低分（如 20分）。
- **步骤 4.2：接口封装 (API Integration)**
  - **思路：** 将上述所有流程封装为一个标准化的服务接口。当 TCM-Agent 的主流程请求验证“Si-ni-San 对肝纤维化的机制”时，该模块在后台静默运行这套复杂的多智能体工作流，最后只返回极其干净、带有置信度评分的 JSON 证据矩阵。
