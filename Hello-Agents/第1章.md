# 第一章 初识智能体

## 1.1 什么是智能体？

在人工智能领域，智能体被定义为任何能够通过**传感器（Sensors）**感知其所处**环境（Environment）**，并**自主**地通过**执行器（Actuators）**采取**行动（Action）**以达成特定目标的实体。

### 1.2.2 智能体的运行机制

智能体并非一次性完成任务，而是通过一个持续的循环与环境进行交互，这个核心机制被称为 **智能体循环 (Agent Loop)**。

![图片描述](https://raw.githubusercontent.com/datawhalechina/Hello-Agents/main/docs/images/1-figures/1757242319667-5.png)

行动并非循环的终点。智能体的行动会引起**环境 (Environment)** 的**状态变化 (State Change)**，环境随即会产生一个新的**观察 (Observation)** 作为结果反馈。这个新的观察又会在下一轮循环中被智能体的感知系统捕获，形成一个持续的“感知-思考-行动-观察”的闭环。智能体正是通过不断重复这一循环，逐步推进任务，从初始状态向目标状态演进。

### 1.4.2 作为自主协助者的智能体

从早期的BabyAGI,AutoGPT,到如今更为成熟的CrewAI,AutoGen,MetaGPT,LangGraph等优秀框架，它们的架构范式大致可以归纳为几个主流方向：

1. **单智能体自主循环**：这是早期的典型范式，如 **AgentGPT** 所代表的模式。其核心是一个通用智能体通过“思考-规划-执行-反思”的闭环，不断进行自我提示和迭代，以完成一个开放式的高层级目标。

2. **多智能体协作**：这是当前最主流的探索方向，旨在通过模拟人类团队的协作模式来解决复杂问题。它又可细分为不同模式： **角色扮演式对话**：如 **CAMEL** 框架，通过为两个智能体（例如，“程序员”和“产品经理”）设定明确的角色和沟通协议，让它们在一个结构化的对话中协同完成任务。 **组织化工作流**：如 **MetaGPT** 和 **CrewAI**，它们模拟一个分工明确的“虚拟团队”（如软件公司或咨询小组）。每个智能体都有预设的职责和工作流程（SOP），通过层级化或顺序化的方式协作，产出高质量的复杂成果（如完整的代码库或研究报告）。**AutoGen** 和 **AgentScope** 则提供了更灵活的对话模式，允许开发者自定义智能体间的复杂交互网络。

3. **高级控制流架构**：诸如 **LangGraph** 等框架，则更侧重于为智能体提供更强大的底层工程基础。它将智能体的执行过程建模为状态图（State Graph），从而能更灵活、更可靠地实现循环、分支、回溯以及人工介入等复杂流程。

### 1.4.3 Workflow和Agent的差异

**Workflow 是让 AI 按部就班地执行指令，而 Agent 则是赋予 AI 自由度去自主达成目标。**

![图片描述](https://raw.githubusercontent.com/datawhalechina/Hello-Agents/main/docs/images/1-figures/1757242319667-18.png)

**这种基于实时信息进行动态推理和决策的能力，正是 Agent 的核心价值所在。**



## 思考

### 科研场景的多智能体协作框架未来会如何发展？

结论：成长为“面向科研闭环的科学工作流操作系统”，中心会从“对话"转向”研究流程“，从“单次回答”到”可追踪、可复现、可审计的持续研究“。



**第一，科研多智能体会从“文献综述型”走向“科研闭环型”。**
 早期系统最强的是并行检索、归纳和信息综合。Anthropic 明确提到，多智能体在需要同时沿多个方向展开的 breadth-first research 上优势很明显；FutureHouse 的 PaperQA2 已经把“找论文—抽信息—沿引用图追踪—形成答案”做成了专门能力。下一步则是把这些能力继续向后接到假设生成、实验设计、数据分析、图表生成、论文写作与复核上。Robin 已经把文献检索代理和数据分析代理接成端到端发现流程；AI Scientist-v2 则把假设、实验、写作、审稿式改进串成了一个 agentic tree search 系统。

**第二，框架核心会从“多角色聊天”转向“显式工作流 + 状态机 + 长任务运行时”。**
 科研任务天然是长周期的：要跨多天、多轮分析、等待外部结果、经常中断后继续。所以真正适合科研的框架，核心能力不是群聊，而是持久化状态、恢复执行、人工插点、版本管理。LangGraph 把 durable execution、human-in-the-loop、memory 放在核心位置；Microsoft Agent Framework 也把 workflows 明确定位为把 AI agent 和业务流程融合起来的编排层。科研型框架会越来越像这个方向，只是把“业务流程”换成“研究流程”。

**第三，Agent 会越来越岗位化，而不是通用化。**
 未来更稳的科研系统，不会是一个万能科学家 Agent，而是“文献 Agent、假设 Agent、实验设计 Agent、代码/统计 Agent、审稿人 Agent、仪器操作 Agent”这样的岗位协作。原因很简单：不同环节需要的工具、上下文和评价标准完全不同。PaperQA2 之所以强，是因为它把自己收窄在科学文献检索与综合；Anthropic 的经验也表明，多智能体最值钱的是并行分解和专门化处理。也就是说，**科研框架未来的关键词不是更多 Agent，而是更清晰的岗位边界和交接协议。**

**第四，真实科研框架一定会往“lab-in-the-loop”发展，而不只停留在屏幕里。**
 这是科研场景和普通信息检索场景最大的差别。Nature Communications 的 AILA 工作已经说明，LLM agents 可以用于自动化 AFM 这类真实实验设备；相关综述与论文也在不断把“scientific instruments + agent”推向更强的闭环。换句话说，未来的科研多智能体框架，不只是会读论文、写代码，还会逐步接入 LIMS/ELN、实验设备、机器人平台和数据采集系统。谁能把“数字工作流”真正接到“物理实验流”，谁就更接近科研生产力平台。

**第五，协议标准化会决定科研多智能体能不能真正落地。**
 科研环境的异构性太强：论文数据库、私有知识库、代码仓库、实验记录、统计软件、仪器控制接口都不一样。这里未来大概率会形成分层：MCP 负责把外部工具和数据源标准化接进来，A2A 负责不同 agent 之间跨框架、跨厂商协作。Google 在发布 A2A 时就明确说它补充的是 MCP，而 MCP 本身就是为模型接入外部数据和工具而设计的开放协议。对科研来说，这意味着未来框架竞争不只是“谁更聪明”，还包括“谁更能接系统、接设备、接团队”。

**第六，评测与治理会从附属品变成主干。**
 FutureHouse 推出 BixBench 的直接原因，就是现有 benchmark 很难评估真实的开放式科研分析工作流；Nature Communications 关于 AI scientists 风险的文章则明确指出，这类系统在自主实验和科研决策中会带来新的安全与误用风险；Nature 2026 的评论也强调，目标不该是“完全自动化”，而应是保留创造力、责任与惊喜的科研平台。科研框架未来不会只比“回答像不像人”，而会比 **证据链是否完整、实验建议是否可复现、过程是否可追踪、风险是否可控、人与系统的责任边界是否清楚**。



简单概况：**科研多智能体框架的未来 = 研究闭环工作流化 + 岗位专门化 + 实验系统接入 + 证据/溯源优先 + 协议互联 + 人类主导下的治理。**
